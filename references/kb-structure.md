# Knowledge Base Structure

## Paths

**Personal KB Root:** `./knowledge/` (current working directory + /knowledge)

**Project KB Root:** `./docs/knowledge/` (current working directory + /docs/knowledge)

## KB Location Detection

Skills automatically detect which KB to use by checking convention paths for `.index/summaries.json`:

1. Check `./docs/knowledge/`
2. Check `./knowledge/`
3. Check `./`

**If multiple found:**

- Show list with document counts
- Prompt user to choose

**If none found:**

- Prompt: "Create KB at [1] ./docs/knowledge/ or [2] ./knowledge/? [1-2]:"
- Prompt: "Is this a project KB (fixed structure) or personal KB? [project/personal]:"
- Save `kb_type` in metadata.json

**KB Types:**

- **Project KB**: Summaries in `.index/`, content at source paths. AI-generated dynamic categories.
- **Personal KB**: Full content stored in KB documents. AI-generated dynamic categories.

**Directory Layout (Personal KB):**

```text
./knowledge/
  .index/
    summaries.json    # Document metadata and search index
    metadata.json     # Aggregate statistics (kb_type: "personal")
    clusters.json     # Generated by rebuild
    clusters/         # Cluster details
  <category>/
    <document>.md     # Markdown documents with frontmatter
```

**Example (Personal KB):**

```text
./knowledge/
  .index/
    summaries.json
    metadata.json
  web-development/
    react-hooks.md
    nextjs-routing.md
  research/
    ai-paper-2024.md
```

**Example (Project KB):**

```text
./docs/knowledge/
  .index/
    summaries.json
    metadata.json (kb_type: "project")
  architecture/
    system-design.md
    tech-stack.md
  api/
    endpoints.md
    authentication.md
  deployment/
    ci-cd-pipeline.md
```

## Document Format

Each document is a markdown file with YAML frontmatter:

```markdown
---
title: Document Title
tags: [tag1, tag2, tag3]
summary: Brief summary of content
key_concepts: [concept1, concept2]
related_topics: [topic1, topic2]
category: category-name
---

# Document content in markdown...
```

## Category Organization

Both Personal and Project KB use AI-generated dynamic categories:

- Lowercase with hyphens (e.g., `web-development`, `machine-learning`, `architecture`)
- Suggested by AI during ingestion based on content analysis
- User-editable
- Rebuild groups related documents into clusters by shared themes

## Source Field (Project KB Only)

The Project KB stores summaries in `.index/summaries.json` and references original content via the `source` field.
This enables progressive disclosure without duplicating content.

**Source Field Behavior:**

- Points to original file location relative to KB root
- Automatically recovered if source file is moved (via neat-knowledge-query or neat-knowledge-rebuild)
- Optional `broken_link: true` flag if automatic recovery fails
- Used only for Project KB; Personal KB omits source field (content stored in KB document)

See [references/kb-schema.md](kb-schema.md) for detailed schema.

## Filename Convention

Filenames are:

- Sanitized from document title
- Lowercase, spaces replaced with hyphens
- Special characters removed
- Extension: `.md`

Examples:

- "React Hooks Guide" → `react-hooks-guide.md`
- "AI & Machine Learning" → `ai-machine-learning.md`
